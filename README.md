# LegalPulse

Agentic AI business-development intelligence platform for legal teams, built to the Octave brand.

LegalPulse continuously surfaces business opportunities by mining the firm's own client and matter data, scanning external signals for new prospects, and processing global news for legally-significant events — then producing partner-ready briefings with talking points, suggested partner, urgency tier, and a full source audit trail.

## Quick start

```bash
# 1. Install dependencies (Node 20+; the project uses sql.js, bcryptjs — no native bindings).
cd backend && npm install
cd ../frontend && npm install

# 2. (Optional) Set environment variables for external sources.
#    Without these the bake still produces a meaningful demo using synthetic seed signals.
export COURTLISTENER_API_TOKEN=...
export COMPANIES_HOUSE_API_KEY=...
export SEC_EDGAR_USER_AGENT="LegalPulse Demo (you@example.com)"

# 3. Run the demo bake. Produces data/demo-snapshot.json and data/bake-summary.json.
node scripts/bake-demo.js

# 4. Build the frontend.
cd frontend && npm run build && cd ..

# 5. Start the server. Serves the React build and the Express API on the same port.
cd backend && node server.js
# → http://localhost:8000
```

On Windows / PowerShell, prefix env vars individually:

```powershell
$env:COURTLISTENER_API_TOKEN="..."
$env:COMPANIES_HOUSE_API_KEY="..."
$env:SEC_EDGAR_USER_AGENT="LegalPulse Demo (you@example.com)"
node scripts/bake-demo.js
```

If your machine routes through a TLS-intercepting corporate proxy (JKH-style), prefix the bake and server commands with `NODE_TLS_REJECT_UNAUTHORIZED=0`. Do NOT ship that env var in production — Render does not need it.

## Development

```bash
# Terminal 1 — Express API
```

## Architecture

```
│   ├── seed/                  # Hartwell & Stone seed JSON (firm, clients, prospects, matters, partners, taxonomy, conflicts)
│   ├── demo-snapshot.json     # Output of the bake — committed to the repo
│   └── bake-summary.json      # Bake stats
├── scripts/
│   ├── bake-demo.js           # Demo bake — fetches external signals, classifies, runs engines, writes snapshot
│   └── bake-helpers.js        # Deterministic heuristics used in offline-mode bakes
├── backend/
│   ├── server.js              # Express app — serves API + frontend build
│   ├── database.js            # sql.js wrapper (async init); db stored in %LOCALAPPDATA% in dev
│   ├── routes/                # Express routers (auth, workspaces, opportunities, engines, chat, ingestion)
│   ├── agents/                # Anthropic-backed agents (signal classifier, entity linker, briefing generator, etc.)
│   ├── sources/               # External-source fetchers + orchestrator
│   ├── engines/               # Cross-sell, prospect discovery, event intelligence engines
│   └── lib/                   # Shared utilities (http, audit, ids, entities, conflicts, clustering, dedupe)
└── frontend/
    └── src/
        ├── pages/             # Dashboard, OpportunityDetail, Clients, ClientDetail, Matters, Signals, AuditTrail, Sources, Setup

        ├── styles/            # tokens.css (Octave design tokens) + layout.css
        └── api.js             # Axios client (baseURL: /api)
```

### The demo bake

The bake is the central architectural decision: **the demo opens to a fully populated state, generated by `scripts/bake-demo.js` before deployment.** No external API call is made on the demo viewer's behalf.

`scripts/bake-demo.js` supports two modes:

1. **Online (hybrid).** When `ANTHROPIC_API_KEY` is set, the bake uses the real agents (Sonnet, Haiku, Opus per agent inventory) for the highest-quality output.
2. **Offline (deterministic).** When the API key is absent, the bake uses rule-based heuristics from `bake-helpers.js` to classify signals, pick services, and template briefings. The snapshot is still populated and traceable; briefings are coarser.

Both modes call the real external sources (GDELT, EDGAR, CourtListener, Companies House, OFAC SDN, UK OFSI, RSS feeds, sanctions lists). If a fetch fails wholesale, the bake falls back to a built-in set of 20 synthetic-but-realistic seed signals (Hormuz, Red Sea, EU AI Act, FCA enforcement, etc.) so the demo never opens empty.

Re-bake at any time by re-running `node scripts/bake-demo.js`.

### Sources

- **GDELT** — global news (no auth)
- **SEC EDGAR** — US securities filings (User-Agent required)
- **CourtListener** — US federal/state courts (free API token)
- **Companies House** — UK company data (free API key)
- **Federal Register** — US regulations (no auth)
- **OFAC SDN / UK OFSI / EU Open Data sanctions** (no auth)
- **DOJ / FTC / DG COMP / FCA / Lexology / JD Supra** — RSS feeds (no auth)
- **EUR-Lex** — EU regulations (RSS)
- **Career-page scraping** — GC/CCO hiring signals

### BYOK

The Anthropic API key is bring-your-own — set in Settings (or via `auth.setApiKey`). Stored in the workspace's user record. Required for live chat, custom analyses, briefing/email regeneration, and User Input Mode ingestion. The pre-baked demo snapshot does NOT require a key to display.

## Deployment (Render.com)

1. Push the repo to GitHub.
2. Create a new Web Service on Render, connect the repo, select "Use render.yaml" or paste:
   - Build: `cd frontend && npm install && npm run build && cd ../backend && npm install`
   - Start: `cd backend && node server.js`
3. Set environment variables in the Render dashboard:
   - `COURTLISTENER_API_TOKEN`
   - `COMPANIES_HOUSE_API_KEY`
   - `SEC_EDGAR_USER_AGENT`
   - (Do **not** set `ANTHROPIC_API_KEY` at the platform level — it is per-user BYOK.)
4. Deploy. The first request to `/` loads the React build; the demo workspace auto-creates from `data/demo-snapshot.json`.

The `render.yaml` at the repo root mirrors all of the above for IaC-style provisioning.

## Octave brand compliance

Every view uses the Octave light-mode-first variant: white canvas with black panels and a single turquoise accent. Design tokens live in `frontend/src/styles/tokens.css` and are referenced from every component. **No** pink, **no** gradients, **no** drop shadows on the logo, **no** hardcoded hex values in component styles. Typography uses ITC Avant Garde Gothic Pro (display) and Lato (body) with web fallbacks if the licensed font is not available.

## Build environment notes (JKH machines)

- Node.js v20+ runtime — Python is not used.
- All npm packages are pure JS — `sql.js` (not `better-sqlite3`), `bcryptjs` (not `bcrypt`), `@napi-rs/canvas` if needed (not `canvas`).
- For corporate TLS interception, set `NODE_TLS_REJECT_UNAUTHORIZED=0` at the npm and node level. Do **not** ship this to production.
- Database is stored in `%LOCALAPPDATA%\LegalPulse\legalpulse.db` in development (outside OneDrive sync) and in `backend/storage/` in production.
